Part 1

Compare the three ways to implement animations

Velocity.js？CSS transition ？jQuery animation

jQuery animation

1、layout thrashing will cause stutters of animation at the beginning.

2、it has large memory consumption, and it always triggers garbage collection, causing stutters of animation in the running.

3、it's not use RAF.

CSS transition

1、Avoiding memory consumption to reduce the stutters.

2、Using a mechanism similar to RAF.

3、Hardware speedup.

4、The browser below IE10（ IE8、IE9 ）does not support transition, and it can't be completely controlled by JS

Velocity.js

A new animation engine, mading a lot of optimizations in this area:

1、Fine tuning the stack to minimize the layout thrashing.

2、Cache the property values in the chain operation to minimize the DOM query operation.

Other advantages of Velocity.js

1、More lightweight, greatly improving the performance and workflow of UI animation.

2、free and easy to use.
